# 광주광역사고조사센터 대시보드 개발 규칙 (Development Rules)

본 문서는 향후 유지보수 및 AI 에이전트와의 협업 시 컨텍스트 유실(Context Loss)을 방지하고 확립된 로직(Regression 방지)을 유지하기 위한 핵심 개발 및 아키텍처 규칙을 정의합니다.

## 1. 데이터 로딩 및 브라우저 캐싱 (Data Loading & Caching)
- **로컬 스토리지 캐싱 필수:** `getNews_api02` 공공데이터 API 통신 시 발생하는 로딩 병목을 해결하기 위해 무조건 `localStorage` 클라이언트 사이드 캐싱을 운용한다.
- **오전 9시 갱신 룰 (TTL):** 지정된 만료 시간(ex: 4시간)을 쓰지 않고, "저장된 시간이 오늘(또는 어제) 오전 9시 이전인가?"를 판별(Effective 9 AM 로직)하여 매일 9시 정각을 기준으로 자동 갱신되도록 한다.
- **메모리 초과(QuotaExceeded) 방지 및 용량 다이어트:** 
  - 공공 API 응답 시 원본 그대로 캐시에 밀어 넣지 말고, 반드시 `contents` 등에서 정규식(`/ <[^>]*>?/gm ` 등)을 활용해 **HTML 태그를 완전히 제거하고 순수 텍스트만 추출·압축**하여 저장한다. (11MB -> 0.4MB 경량화)
  - `localStorage.setItem()` 수행 전 반드시 `removeItem()`을 통해 과거 사용하던 구버전 캐시 키들(`v1`, `v2` 등)을 명시적으로 지운다.
- **예외 처리:** 만약 로컬 스토리지가 꽉 차서 용량을 초과하거나 캐싱 저장이 실패하더라도, 예외(`try-catch`) 처리하여 UI 그리기(렌더링)는 멈추지 않고 수행되도록 방어 코드를 구비한다.

## 2. UI 렌더링 및 Chart.js 정책 (UI & Chart Guidelines)
- **차트 중첩 렌더링(Canvas in use) 방지:** 캐시된 상황이나 검색어 필터링 조작 등 빈번한 호출 과정에서, `new Chart(...)`를 생성하기 전 **반드시 `Chart.getChart(ctx)`로 기존 인스턴스를 확인하고 `.destroy()`하여 메모리를 비운 뒤**에 새로 그려야 한다.
- **반응형 가로 막대 그래프 전환:** 모바일(화면 너비 768px 미만) 환경에서는 통계 차트의 `options.indexAxis`를 `'y'`로 동적 할당하여, 좁은 화면 내 라벨(지역명) 겹침을 방지하고 가독성을 확보한다.
- **미니멀리즘 원칙:** "N건 검색됨", "방금 전 업데이트됨" 과 같은 기술적인 부가 텍스트UI를 전면 배치하지 말고 백그라운드로 돌려 화면의 심미성과 프리미엄 스타일(Premium UI)을 유지한다.

## 3. 데이터 파싱 및 정규화 (Parsing & Extracting Engine)
- **검색 및 지역 누락 방지:** API 검색 결과의 `keyword`(제목)만 가지고 의존하지 않는다. `contents`(본문)에 명시된 키워드까지 정규식을 통해 파싱(`_parsed` 객체 활용)하여 광주 등 특정 지역 누락 사고를 방지한다.
- **전국 지표 제공 의무:** 비교 그래프 작성 시에는 반드시 전체 조회 건수를 총합하여 '전국'이라는 기준선 레이블을 최상단에 제공해야 한다.
