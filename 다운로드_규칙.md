# 프로젝트 개발 규칙 (웹 애플리케이션 다운로드 기능)

## Google Drive 직접 다운로드 링크 구현 규칙

Google Drive에 업로드된 파일(PDF, HWP, 엑셀 등)을 웹사이트에서 '단일 클릭 직접 다운로드' 방식으로 제공할 때 발생하는 브라우저 오류(UUID로 파일명이 깨지는 현상 등)를 방지하기 위해 다음 규칙을 반드시 엄수한다.

### 1. 허용되는 URL 포맷
구글 드라이브 파일의 공유 링크(`.../view?usp=sharing`)를 그대로 사용하지 말고, 반드시 아래의 **직접 다운로드 전용 라우팅(`uc?export=download`) 포맷**으로 변환하여 사용해야 한다.

- ❌ 금지된 포맷 (미리보기 권장용으로 직접 다운로드 시 오류 발생 가능)
  `https://drive.google.com/file/d/[파일ID]/view?usp=sharing`
  `https://docs.google.com/document/d/[파일ID]/edit`
  
- ✅ **필수 적용 포맷 (Direct Download)**
  `https://drive.google.com/uc?export=download&id=[파일ID]`

### 2. HTML 태그 속성 제어 (엄수)
크롬, 엣지 등 모던 브라우저에서 Cross-Origin(외부 도메인) 다운로드 시 충돌이나 파일명 변조(UUID 발생)를 막기 위해 다음 속성 규칙을 적용한다.

1. **`download` 속성 절대 사용 금지**:
   - `download="파일명.pdf"`와 같이 HTML5 다운로드 속성을 강제로 부여하면, 외부 드라이브 서버(Google)의 Content-Disposition 응답과 충돌하여 파일명이 알 수 없는 영문/숫자(UUID)로 변조된다. 브라우저가 원본 파일명을 인식하도록 `download` 속성을 완전히 제거해야 한다.
2. **`target="_blank"` 속성 지양**:
   - 새 창을 띄울 필요 없이 현재 탭에서 파일 다운로드만 부드럽게 백그라운드로 진행되도록 `target="_blank"`를 사용하지 않는다.

**적용 예시 코드:**
```html
<!-- 올바른 직접 다운로드 링크 예시 (PDF 파일인 경우) -->
<a href="https://drive.google.com/uc?export=download&id=11D7iChWyiRVt7UfWOgfXhHduCeES5FJJ" class="btn-download">
  <i data-lucide="download"></i> 📄 PDF 서식 다운로드
</a>

<!-- 올바른 직접 다운로드 링크 예시 (HWP 한글 파일인 경우) -->
<a href="https://drive.google.com/uc?export=download&id=1cONtV7X-l1s7PXrg3xB2LA_hsspNtkBj" class="btn-download">
  <i data-lucide="download"></i> 📄 HWP 서식 다운로드
</a>
```

### 3. 디버깅 및 캐시 주의사항
링크가 올바르게 수정되었음에도 사용자가 계속 오류를 겪거나 다운로드 주소가 갱신되지 않는다면, **100% 브라우저 하드 캐시(Hard Cache)** 또는 **로컬 에디터의 덮어쓰기** 문제이다. 이 경우 사용자에게 다음을 강력히 안내한다.
- 닫혀있지 않은 기존 에러 탭(Google Docs 등)을 모두 강제 종료.
- 대상 페이지에서 **강력 새로고침 (Ctrl+F5 또는 Shift+F5)** 후 재진입.
